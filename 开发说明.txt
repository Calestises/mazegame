迷宫游戏开发说明

1. 项目概览

本项目是一个使用 C 语言编写的终端迷宫游戏，当前主程序源码为 mazegame.c。

当前版本的核心特征：
- 随机生成迷宫，不是固定地图
- 共 3 个关卡，后续关卡地图更大
- 玩家需要收集钥匙后才能进入门
- 地图中包含金币与传送门
- 支持 3 个固定存档槽
- Windows 下支持 UTF-8 中文显示、颜色显示、较平滑的重绘

当前版本已经移除的设计：
- 敌人系统
- 陷阱系统

3. 核心常量与结构

3.1 宏定义

位于 mazegame.c 第 12 行到第 18 行：
- LEVEL_COUNT：关卡总数
- MAX_ROWS：地图最大行数
- MAX_COLS：地图最大列数
- MAX_TELEPORTS：最大传送点数量
- SAVE_SLOT_COUNT：存档槽数量
- SAVE_FILE_PREFIX / SAVE_FILE_SUFFIX：存档文件名前后缀

3.2 Point

位于 mazegame.c 第 20 行。
用于表示一个地图坐标：

typedef struct {
    int row;
    int col;
} Point;

3.3 LevelConfig

位于 mazegame.c 第 25 行。
用于定义每一关的固定参数：

typedef struct {
    int rows;
    int cols;
    int keyTotal;
    int goldTotal;
    int teleportPairs;
} LevelConfig;

3.4 GameState

位于 mazegame.c 第 33 行。
这是整个游戏的运行状态中心，保存当前关卡、地图大小、玩家位置、金币与钥匙数量、步数、地图数组、传送门坐标、消息栏文本。

4. 当前关卡配置

5.2 地图生成模块

主要函数：
- initialize_map()

- count_open_neighbors()
- find_farthest_point()

关键思路：
- 先把地图填满墙
- 用随机 DFS 回溯挖出道路
- 用 BFS 算起点到所有格子的距离
- 门放在最远点
- 钥匙优先放死路尽头

道具放置逻辑可以从 mazegame.c 第 523 行开始看。

5.6 主循环

main() 位于 mazegame.c 第 871 行。

主流程：
1. 初始化控制台

7. 处理移动、保存、读档、退出
8. 显示胜利或结束信息

6. 当前存档机制

6.1 存档槽设计

当前不是一个固定 save.txt，而是固定 3 个槽：
- save1.txt
- save2.txt
- save3.txt

6.2 保存方式

游戏中按 p：
1. 程序会先询问保存到几号槽
2. 输入 1、2、3
3. 保存后消息栏显示“已保存到 X 号存档”

6.3 读取方式

两种方式：
- 主菜单选“读取存档”后再选槽位
- 游戏中按 o 后再选槽位

6.4 存档格式

save_game() 和 load_game() 仍使用纯文本格式。

第一行保存 13 个整数：
currentLevel rows cols hp gold keyCount requiredKeys steps playerRow playerCol startRow startCol teleportCount

然后是：
- rows 行地图
- teleportCount 行传送门坐标

如果修改了持久化字段，必须同步改 save_game() 和 load_game()。

7. 编译与运行

推荐命令：
gcc -Wall -Wextra -std=c11 -finput-charset=UTF-8 -o mazegame.exe mazegame.c

如果只是运行游戏，直接双击 mazegame.exe 即可。

8. 编码与终端要求

当前代码包含中文文本和特殊字符 █，建议：
- 源文件保存为 UTF-8
- 编译时显式加 -finput-charset=UTF-8
- 尽量在支持 UTF-8 的终端运行

Windows 下还会调用：
- SetConsoleOutputCP(CP_UTF8)
- SetConsoleCP(CP_UTF8)
- setlocale(LC_ALL, ".UTF-8")

9. 已知限制

- 当前仍是终端字符游戏，不是图形窗口程序
- hp 字段保留，但玩法中没有复杂生命系统
- 只支持固定 3 个存档槽
- 手动修改存档内容可能导致读档失败
- 不同终端对颜色和刷新效果的表现可能略有区别

10. 后续扩展建议

适合继续扩展的方向：
- 增加更多关卡
- 提高后期关卡的金币、钥匙、传送门配置
- 新增计时或评分系统
- 新增特殊门、特殊钥匙
- 把 3 个存档槽扩展成动态存档列表
- 以后如要恢复敌人或陷阱，建议重新设计，不建议直接回滚旧实现

11. 修改后最低测试清单

每次改动后至少验证：
1. 程序能正常启动
2. 中文显示正常
3. w/a/s/d 移动正常
4. 钥匙、金币、传送门逻辑正常
5. 撞墙与开门提示正常
6. p 保存时可以选择槽位
7. o 读档时可以选择槽位
8. save1.txt 到 save3.txt 生成与读取正常
9. 通关与最终胜利提示正常
