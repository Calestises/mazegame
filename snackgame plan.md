# 迷宫游戏项目计划

## 1. 目标

用 C 语言完成一个字符版迷宫游戏，满足以下要求：

- `w/a/s/d` 控制玩家移动
- 一共 3 个关卡
- 包含钥匙、金币、陷阱、传送点、敌人
- 拿到钥匙后到达门口即可通关
- 玩家有生命值，被敌人或陷阱伤害后扣血
- 支持保存和读取进度

## 2. 推荐开发顺序

### 第一阶段：先跑通最小版本

先只做这些功能：

- 地图显示
- 玩家移动
- 墙体碰撞
- 门和钥匙
- 通关切换下一关

你要先确认一件事：角色能不能稳定在地图里移动，不能一开始就把敌人、存档、机关全堆进去。

### 第二阶段：加入道具和机关

补上下面几个元素：

- 金币：捡到后 `gold++`
- 陷阱：踩中后 `hp--`
- 传送点：把玩家传到另一处

这一阶段的重点是“玩家移动到格子后，如何根据格子字符触发不同效果”。

### 第三阶段：加入敌人

推荐用 3 种不同规则，分别对应 3 个关卡：

- 第 1 关：水平巡逻敌人
- 第 2 关：垂直巡逻敌人
- 第 3 关：追踪型敌人

每次玩家行动一次，敌人也行动一次。这样最容易理解，也最容易调试。

### 第四阶段：做存档

至少保存这些数据：

- 当前关卡
- 玩家坐标
- 起点坐标
- 生命值
- 金币数
- 是否有钥匙
- 步数
- 当前地图内容
- 敌人位置和方向
- 传送点位置

存档建议用文本文件 `save.txt`，练习 `fopen / fprintf / fscanf / fclose`。

### 第五阶段：测试和修补

重点测这几类问题：

- 撞墙时会不会穿墙
- 没钥匙时能不能错误开门
- 捡过的金币和钥匙会不会重复出现
- 传送后会不会落到错误位置
- 敌人碰到玩家时是否正确扣血
- 存档后再读档，地图状态是否恢复正确

## 3. 代码结构建议

建议你重点理解这几个函数：

- `load_level()`：加载关卡
- `draw_game()`：绘制地图
- `process_move()`：处理玩家移动
- `handle_special_tile()`：处理钥匙、金币、陷阱、传送、门
- `move_enemies()`：处理敌人移动
- `save_game()`：保存进度
- `load_game()`：读取进度

## 4. 你现在可以直接做什么

先运行当前代码版本，确认基础流程，再尝试自己改下面 3 件事：

1. 把地图换成你自己的布局
2. 新增一种道具，比如回血包 `M`
3. 给敌人增加更复杂的移动规则

## 5. 编译运行

如果你本机装了 `gcc`，可以用：

```bash
gcc maze_game.c -o maze_game
./maze_game
```

如果你用的是 Windows 的 MinGW，通常是：

```bash
gcc maze_game.c -o maze_game.exe
maze_game.exe
```
